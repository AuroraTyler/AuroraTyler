Same Page Errors

What:

Placing error messages for a form on the same page as the form’s input area.

When:

Use so that when a user enters information that isn’t valid or acceptable, they know right away and can fix it before it becomes a problem. It can also help a user understand what is needed if they are unsure what to enter.

Why:

Old error messages showed up in popup dialog boxes, that had to be closed before a user could adjust their input, meaning that they wouldn’t be able to read the message anymore. Other error messages would show up on a separate page, so the message was still limited to 1 reading. Having the message next to the input now means that the user can see more easily where the problem is.

How:

Design the form so that certain types of errors aren’t possible. For example, using dropdowns instead of text fields. Mark all required fields, but don’t make too many fields necessary. When errors happen place messages at the top of the form with graphics and bold or red text. Also, mark the fields where the errors occurred.

The name is required.

If you click on the name field, but do not enter anything, you get an error

If you enter text into the email field that doesn't follow the correct format, you get an error. The correct format is: me@example.com
