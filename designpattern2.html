Design Pattern 2: Prominent "Done" Button

What do you want to know?

What is the "Done" button?
When do you use the "Done" button?
Why use the "Done" button?
How to use the "Done" button?

Answers

What:

The final button that a user clicks in the process of completing their task is big, is at the end of the visual flow and is clearly labeled.

When:

Any button that saves work, submits information, or otherwise completes some set of actions by the user should be prominent.

Why:

By making the last step stand out, the user isn’t left wondering or confused if the process is over. They know that their work is saved, the information is submitted, etc.

How:

Placement, appearance and labeling are the keys to a well-done button. The button should appear at the logical conclusion of the set of actions that it is finishing, such as the end of the page or the button of the form. It should look like a button with colors that contrast from the background and has well defined edges. The button should always be labeled, and the more specific the better. Don’t use an icon because that can be ambiguous. Use text that sums up the action that the button does, “Submit”, “Save”, etc.
